  • пример отправки общей ошибки с сервера и ошибок для каждого поля

    POST: /api/xxx send data: 
    {
    phone: x,
    password: x
}
    accept data: 
    {
    success: false,
    data: null,
    errors: {
        common: {
            error: 'Неверные телефон/пароль',
            fields: ['phone', 'password']
        },
        fields: [
            {
                name: 'phone',
                error: 'Неверный номер телефона'
            },
            {
                name: 'email',
                error: 'Такая почта уже зарегистрирована'
            }
        ]
    }
}

  • первый шаг регистрации. нужно только проверить правильность данных на сервере, отравить проверочный код на телефонный номер и в случае чего вернуть ошибки валидации

    POST: /api/register_one send data: 
    {
    imgId: x,
    name: x,
    sername: x,
    birthday: x (dd MMM yyyy),
    sportTime: x (0->7),
    phone: x,
    gender: x (male, female)
}
    accept data: 
    {
    success: true,
    data: null,
    errors: null
}

  • загрузка видео с ютуб. отправляется ссылка на видео youtube, в базе создается запись с собственным id, с remoteId этого видео (айдишник видео на ютубе) и картинкой. Эти же данные возвращаются при "загрузке видео"

    GET: /api/youtube send data: 
    {
    link: xxx
}
    accept data: 
    {
    success: true,
    data: [
        id: xxx,
        remoteId: xxx,
        img: xxx
    ],
    errors: null
}
